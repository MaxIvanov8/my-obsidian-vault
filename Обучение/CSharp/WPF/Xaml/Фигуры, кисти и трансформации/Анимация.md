Каждая анимация в WPF работает на основе отдельного свойства зависимости. 

Для того, чтобы анимировать свойство, требуется класс анимации, который поддерживает тип данных свойства. Например, для анимации свойства Height типа Double следует использовать класс DoubleAnimation. 

**Стандартная частота анимации** - 60 кадров в секунду.

# Типы анимации
1. Линейная интерполяция – последовательное изменения значения свойства (такие классы называются по принципу <ИмяТипа>Animation, например, ColorAnimation, DoubleAnimation) 
2. Анимация ключевого кадра – изменения свойств в определенный момент времени (такие классы называются <ИмяТипа>AnimationUsingKeyFrames, например, DoubleAnimationUsingKeyFrames) 
3. Анимация на основе пути – изменения значения свойства на основе фигуры, описанной в объекте PathGeometry

# Свойства объектов анимации
1. From – начальное значение, с которого начнется анимация. 
2. To – значение анимированного свойства, до которого будет работать анимация. 
3. By – для создания анимации, которая изменяет значение свойства на определенную величину. 
4. Duration – продолжительность анимации. Задается с помощью объекта Duration

# Свойства TimeLine
TimeLine – базовый класс для всех объектов анимации
1. BeginTime – задержка перед запуском анимации. 
2. Duration – продолжительность 
3. SpeedRatio – изначально равно 1. Чем больше значение, тем быстрее проиграется видео, например, если значение 2, то видео закончится в два раза быстрее. 
4. AccelerationRatio и DecelerationRatio – Делает скорость воспроизведения анимации не линейной. Ускорение и замедление при анимации. Значения от 0 до 1. 
5. AutoRevers – анимация будет запущена в обратном порядке, как только завершится. 
6. FillBehavior – определяет, что случится по завершению анимации. Оставить значения свойств зафиксированными HoldEnd или вернуть к исходным Stop. 
7. RepeatBehavior – позволяет повторить анимацию заданное количество раз, в течении указанного интервала времени

# Анимация ключевого кадра
Анимация ключевого кадра состоит из множества коротких сегментов. 

1. LinerPointKeyFrame – линейный ключевой кадр, при переходе от ключа к ключу выполняется линейная интерполяция для анимируемых свойств. 
2. DiscretePointKeyFrame – при переходе между кадрами дискретного ключа интерполяция не производится.
