Обобщения (generics) - это механизм, поддерживаемый средой CLR и языками программирования для многократного использования кода, а именно - многократного использования алгоритмов.

**Польза обобщений:**
1. Устраняют дублирование кода: вместо `IntBox`, `StringBox` один `Box<T>`.
2. Обеспечивают безопасность типов. В отличие от `ArrayList`, `IComparable`, где все хранится как object. В `List<T>` нельзя положить объект другого типа.
3. Ошибки ловятся на этапе компиляции, а не во время выполнения.
4. Повышают производительность - нет упаковки/распаковки для значимых типов. Поэтому `List<T>` работает быстрее, чем `ArrayList`.

# **Открытые и закрытые типы:**
**Открытый тип** - тип, у которого есть неопределенные параметры типа(`List<T>`, `Dictionary<TKey, TValue>`)

**Закрытый тип** - тип, у которого все параметры типа определены конкретными типами (`List<int>`, `Dictionary<string, int>`)

В райнтайме нельзя создавать объекты открытого типа. 


# Синтаксис
Сущности, для которых возможно использование обобщенного типа:
1. Делегаты
2. Методы
3. Классы
4. Интерфейсы

**Правила описания обобщений в любых сущностях:**
1. После имени сущности указываются параметры `<T>`.
2. После указания параметра могут быть ограничения с маркером where. 
3. Могут содержать ключевые слова применимые к параметрам in/out. Поддерживаются только в интерфейсах и делегатах.



# Встроенные обобщенные делегаты
**С# уже содержит готовые generic-делегаты:**
1. `Func<T, TResult>`- для методов с возвращаемым значением. 
2. `Action<T>` — для методов void.
3. `Predicate<T>` — для методов, возвращающих bool.
Примеры:
`Func<int, int> square = x => х * x`
`Action<string> print = Console.WriteLine`
`Predicate<int> isPositive = x => x > 0`



# Ограничения (Constraints)
Ограничения позволяют указать, какие требования должен удовлетворять тип-параметр T, чтобы его можно было использовать в теле обобщённого класса или метода.
Без ограничений компилятор ничего не знает о Т, кроме того, что он `object` - поэтому доступ к методам и конструкторам будет невозможен без кастов или ограничений.

Виды ограничений:
`where T : class`
`where T : struct`
`where T : new()`
`where T : notnull`
`where T : unmanaged`
`where T : default`


`where T : T : new()`
Указывает, что тип `<T>` должен иметь публичный конструктор без параметров.
```
public class Factory<T> where T new()
{
	public T CreateInstance()
	{
		return new T(); // безопасно
	}
}
```


# Вариантность
Вариантность - это поведение обобщённых типов при преобразованиях между базовыми и производными типами.
Контравариантность и ковариантность - это концепции в обобщениях С#, которые позволяют безопасно преобразовывать типы в обобщённых интерфейсах и делегатах, когда между типами существует отношение наследования.

Она отвечает на вопрос: можно ли присвоить `X<Dog>` в `X<Animal>` или наоборот?


## Виды вариантностей
**Инвариантность** - переменную недопустимо преобразовать к объекту с другим типом в обобщении.
`X<Dog> x = new X<Dog>();`
**Ковариантность (out)** — переменную допустимо преобразовать к объекту с родительским типом в обобщении.
`X<Animal> x = new X<Dog>();`
**Контрвариантность (in)** - переменную допустимо преобразовать к объекту с родительским типом в обобщении.
`X<Dog> x = new X<Animal>();`
